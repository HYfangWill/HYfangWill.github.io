link和@import都是引入CSS的方式

CSS和HTML文档结合的方式

  • 使用元素链接到外部的样式文件
  • <head>内使用<style>
  • @import

  • <body>标签里，使用style属性

    link

  • charset/rev/target 分别是编码/链接文档与当前文档关系/目标页面，HTML5不支持，target属性值是_blank/_self/_top/_parent/frame_name

  • href
    存放链接文件路径
  • hreflang
    链接文件语言
  • media
    显示设备，如screen屏幕、print打印、all所有
  • rel
    当前文档与链接文档的关系，如alternate文档替代版本、author、help帮助文档、icon图标文件、license许可文件、next集合下一个文档、pingback、prefetch、prev集合上一个文档、search搜索文档、sidebar导航文档、stylesheet外部样式表、tag、index索引文档、contents文档目录、bookmark相关文档、chapter文档的章、section文档的节、subseciton文档的小节、appendix文档附录
  • sizes：资源尺寸，只适用于icon资源，属性值为heightxwidth|any，height和width用x和X，any规定图标是可伸缩的
  • type
    链接文档的MIME类型

    @import

link和@import的区别

  • link标签
    HTML和XHTML都有，使网页作者可以增加HTML文档相关的额外信息，额外资源可以引入样式CSS、导航助手、属于另外形式的信息RSS、联系信息等

  • @import
    指定导入的外部样式表及目标设备类型

区别在于：

  1. link是HTML的标签，可被JavaScript控制，@import是CSS的标签，不能被JavaScript改变
  2. link除了调用CSS还可以有其他作用，如声明页面属性、声明目录、RSS等，@import只能调用CSS
  3. link引入的CSS和页面同时加载，@import引入CSS将在页面加载完成后才被加载
  4. @import在CSS2.1才有的语法，在IE5+才能识别，link不存在兼容性问题
  5. link和@import的样式权重和CSS权重计算一致，与代码的顺序有关，可能某些样式被覆盖
  6. 如果JavaScript进行样式选择，用link，因为link是HTML元素，js可以控制
