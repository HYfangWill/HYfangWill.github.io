link和@import都是引入CSS的方式

CSS和HTML文档结合的方式

  • 使用元素链接到外部的样式文件
  • <head>内使用<style>
  • @import

  • <body>标签里，使用style属性

    link

  • charset/rev/target 分别是编码/链接文档与当前文档关系/目标页面，HTML5不支持，target属性值是_blank/_self/_top/_parent/frame_name

  • href
    存放链接文件路径
  • hreflang
    链接文件语言
  • media
    显示设备，如screen屏幕、print打印、all所有
  • rel
    当前文档与链接文档的关系，如alternate文档替代版本、author、help帮助文档、icon图标文件、license许可文件、next集合下一个文档、pingback、prefetch、prev集合上一个文档、search搜索文档、sidebar导航文档、stylesheet外部样式表、tag、index索引文档、contents文档目录、bookmark相关文档、chapter文档的章、section文档的节、subseciton文档的小节、appendix文档附录
  • sizes：资源尺寸，只适用于icon资源，属性值为heightxwidth|any，height和width用x和X，any规定图标是可伸缩的
  • type
    链接文档的MIME类型

    @import

link和@import的区别

  • link标签
    HTML和XHTML都有，使网页作者可以增加HTML文档相关的额外信息，额外资源可以引入样式CSS、导航助手、属于另外形式的信息RSS、联系信息等

  • @import
    指定导入的外部样式表及目标设备类型

区别在于：

  1. link是HTML的标签，可被JavaScript控制，@import是CSS的标签，不能被JavaScript改变
  2. link除了调用CSS还可以有其他作用，如声明页面属性、声明目录、RSS等，@import只能调用CSS
  3. link引入的CSS和页面同时加载，@import引入CSS将在页面加载完成后才被加载
  4. @import在CSS2.1才有的语法，在IE5+才能识别，link不存在兼容性问题
  5. link和@import的样式权重和CSS权重计算一致，与代码的顺序有关，可能某些样式被覆盖
  6. 如果JavaScript进行样式选择，用link，因为link是HTML元素，js可以控制

函数中的()

()运算符，也叫分组运算符，它有多种用法：

  • ()存放函数参数
  • 把函数放在()中，会返回函数本身
  • ()紧跟在函数后面，用于调用函数

关于函数的使用

函数声明

函数声明存在变量提升，并且优先级高于普通变量声明，但同名变量的赋值，后赋值的语句有效

匿名函数

函数表达式

与函数声明不同的是，函数表达式不存在变量提升，并且，值得一提的是，使用函数声明和函数表达式，变量名所指向的内容是完全一致的

立即执行函数IIFE

IIFE（imdiately invoked function expression）：用function定义函数之后，立即调用该函数

function关键字出现在行首，js引擎一律解释为函数声明语句，且函数声明时，js只会解析到大括号就结束，因此不能立即执行，需要在函数外添加

立即执行函数的应用

立即执行函数常用于第三方库，创建一个独立的隔离作用域，避免变量污染（命名冲突），JQuery开篇用的就是立即执行函数，

变量的作用域

JavaScript的变量作用域是特殊的，分为两种

  • 全局变量 - 函数内部可以直接读取
  • 局部变量 - 函数外部不能读取函数内的局部变量，如果函数内部声明变量未用var或其他声明，实际上是声明了全局变量

闭包

基本使用

有时候，需要得到函数内的局部变量，但正常情况下无法获得，就需要使用闭包——定义在一个函数内部的函数

在本质上，闭包就是将函数内部和函数外部连接起来的一座桥梁

f2就是一个闭包，变量n会保存在内存中，不会在调用结束后被回收。

  • 闭包内存消耗会比较大，不能滥用，否则会影响网页性能，IE中甚至导致内存泄漏，因此，可以在退出函数之前，把不使用的局部变量删除
  • 闭包在父函数外部，改变父函数内部变量的值，若父函数是一个对象，闭包作为它的公用方法，把内部变量当做它的私有属性，需要防止随便改变私有属性的值。

this使用

this是函数运行时所在的环境对象

  • 纯粹函数调用
    全局调用时，this为全局对象，而在严格模式下，this会变成undefined
  • 作为对象方法调用
    this指向这个调用者
  • 作为构造函数调用
    使用new关键字时，将实例化一个类，原理是调用类的构造函数，将this绑定到新的对象上
  • apply、call调用
    apply/call接受的第一个参数为对象，this将指向这个对象
  • bind调用
    bind的第一个参数也是对象，this将指向这个对象，与apply/call不同的是，bind绑定时不会执行

CSS盒模型

发表于 2018-09-10 | 分类于 front-end

IE盒模型和W3C盒模型的区别

  • width和height
    IE盒模型content+padding+border，W3C盒模型content

使用

设置盒模型

  • box-sizing可以改变浏览器默认的盒模型
    • content-box：W3C盒模型
    • padding-box：content+padding==width
    • border-box：IE盒模型

对行内元素的设置

  • width/height无效
  • padding-top/padding-bottom/margin-top/margin-bottom无效
  • padding-left/padding-right/margin-left/margin-right有效
  • line-height有效

margin: auto的使用

  • 元素没有设置width和height，会自动填充容器
  • 元素设置了width或height（有效情况下），自动填充会被覆盖，margin为0，因此如果margin设置为auto，可以填充空白，使得元素水平居中

百分比的使用

百分比是相对父元素而言的，这个父元素分为两种：

  1. 如果该元素position没有定位成relative或者absolute，父元素是已经定位的父元素或者是body
  2. 如果该元素position没有定位成relative或者absolute，父元素是直接父元素

具体使用

  • width是相对于父元素的width，height是相对于父元素的height
  • padding/margin不管上下左右，是相对于父元素的width

margin折叠

margin折叠是指毗邻的两个或多个margin会合并为一个margin

规则：

  • 普通流的块元素垂直方向上会折叠
  • 浮动元素或者inline-block或绝对定位的元素不会和垂直方向上的其他元素margin折叠
  • 创建了块级格式化上下文元素，不会和子元素发生margin折叠
  • 元素自身的margin-bottom和margin-bottom相邻也会折叠

计算规则：

  • 正正取大值
  • 正负相加
  • 负负取绝对值较大值

meta标签

发表于 2018-09-10 | 分类于 front-end

HTML可以有多个meta标签，元数据meta是数据的数据信息，meta不会显示在客户端，会被浏览器解析

meta作用

  • name：属性值content所关联的属性名
    • description：描述网页，可用于SEO
    • keywords：关键词，可用于SEO，逗号表示或，空格表示与
    • robot：机器人是否可以检索到这个页面(none不会检测到)
    • author：定义页面作者，content常设置为联系方式
    • application-name：规定页面所代表的 Web 应用程序的名称
    • generator：规定生成文档的软件包
  • http-equiv：属性content
    • content-type：规定文档的字符编码
    • expires：网页过期时不会通过缓存获取网页，content内必须是GMT格式的时间
    • pragma：是否从缓存中访问网页内容，no-cache不能从缓存获取，即不能脱机工作
    • reflesh：刷新时间秒数
    • default-style：预定义的样式
  • charset：字符编码
    • UTF-8：Unicode字符编码
    • ISO-8859-1：拉丁字母表的字符编码
  • scheme：用于翻译content属性的值的方案（HTML5不支持）
    • 定义content属性内的值的格式（或指向一个包含信息的 URI）。

meta在移动端的使用

  • name=”viewport”：移动端屏幕的缩放
    content属性值
    • width
    • height
    • initial-scale
    • minimum-scale
    • maxmum-scale
    • user-scalable
  • name=”format-detection”：是否可以识别电话和邮箱
  • name=”apple-mobile-web-app-capable”：是否开启对web app程序的支持
  • name=”apple-mobile-web-app-status-bar-style”：顶部状态条的颜色
    • white：默认为白色
    • black：黑色
    • black-translucent：灰色半透明

vue双向数据绑定

发表于 2018-09-10 | 分类于 front-end

MVVM

model-view-viewmodel
基于前端开发的架构模式，核心是viewmodel提供对view和model的双向数据绑定

  • 用model渲染view，一旦model变更，重新渲染
  • view的交互改变状态和数据，更新model

    vue.js实现

    数据劫持

      vue采用ES5的Object.defineProperty的getter和setter，结合观察者模式来实现数据绑定。
      当一个JS对象传给Vue实例作为其data选项时，vue会遍历它的属性，用Object.defineProperty转为getter和setter，并添加vue追踪依赖，一旦属性被访问被修改时通知变化。
      vue的绑定过程分为两个步骤：
  1. compile：对目标元素进行解析，识别出所有绑定在元素(通过el属性传入)上的指令
  2. link过程：建立指令与数据(通过data属性传入初始值)的绑定关系，并以数据的初始值进行渲染，订阅Watcher，此时Watcher会将自己添加到消息订阅器(Dep),初始化完毕

    vue双向数据绑定的对象

  • Observer数据监听器
    能够对数据对象的所有属性进行监听(Object.defineProperty实现)，发现变化就通知Watcher订阅者
  • complie指令解析器
    对每个元素节点的指令进行扫描解析，初始化视图，根据指令模板替换数据，绑定Observer和compile的桥梁(Watcher)，可以订阅到每个属性变动的通知，执行指令绑定的相应的回调函数
  • Watcher订阅者
    作为Observer和compile的桥梁，订阅收到的属性变动通知，执行指令绑定的相应的回调函数
  • Dep消息订阅器
    内部维护了一个数组，用来收集Watcher，数据变动触发notify方法，再调用update方法
    graph LR
A[new MVVM] --> B[Observer]
B --> |通知变化| C((Dep))
C --> |通知变化| D[Watcher]
D --> |添加订阅者| C
A --> E[complie]
E --> |订阅数据变化绑定更新函数| D
E --> |初始化视图| F((Update))
D --> |更新视图| F

其他双向数据绑定方式

发布者-订阅者

一般是通过sub、pub的方法实现数据和视图的绑定监听，更新数据方法，vm.set(“property”, value)

脏值检测

通过脏值检测的方式，对所有数据和视图绑定关系进行检测，识别是否有数据发生改变，有变化进行处理，可能引发其他数据的改变，可能会循环多次，知道数据不再变化，将变更的数据更新视图。
angular.js就是使用这种方法，在指定的事件触发时，进入脏值检测，比如

  • DOM事件
  • XHR响应事件
  • 浏览器Location变更事件
  • Timer事件($timeout, $interval)
  • 执行$digest()或$apply()

    Plus

    Object.defineProperty(obj, prop, value)

  • 功能
    直接在目标对象定义一个新属性，或修改现有属性，并返回这个对象
  • 参数
    obj：目标对象
    prop：要定义或者修改的属性
    value：将被定义或修改的属性值
  • 返回
    目标对象
  • 注意
    默认情况下，添加的属性是不可变的

浏览器访问网页

发表于 2018-09-10 | 分类于 front-end

从输入地址到页面显示的流程

graph LR
A[输入URL] --> B[DNS解析URL]
B --> C[TCP连接]
C --> D[浏览器发送请求]
D --> E[服务器响应]
E --> F[关闭连接]
G[浏览器下载文件] --> H[浏览器解析文件]
H --> I[浏览器布局渲染]
  1. 输入URL
  2. DNS解析URL地址中的域名返回IP
    1. 浏览器首先先询问本地hosts文件，检查域名是否有对应的IP
    2. 如果没有对应IP，递归查询本地DNS服务器，DNS服务器迭代查询根、com、www.blog.com的DNS服务器，获得最终的IP
  3. 建立TCP连接
    即经典的TCP三次挥手
    1. 客户端请求连接(SYN数据包)
    2. 服务器确认信息(SYN/ACK数据包)
    3. 客户端确认应答(ACK数据包)
  4. 浏览器发送http请求
    http请求内容：
    • 报文首部：请求行(URI/HTTP版本/请求方法)、请求首部字段(传递重要信息)、通用首部字段、实体首部字段、其他
    • 空行：表示头部结束，不可省略
    • 报文主体
  5. 服务器处理并响应
    响应请求
    • 报文首部：状态行(状态码和信息)、响应首部字段、通用首部字段、实体首部字段、其他
    • 空行：表示头部结束，不可省略
    • 报文主体
  6. 关闭连接
    TCP四次挥手
    1. 客户端请求关闭FIN
    2. 服务器确认请求关闭ACK
    3. 服务器请求关闭FIN
    4. 客户端确认请求关闭ACK
  7. 浏览器解析HTML
    1. 浏览器加载HTML、CSS、JS、图片、视频等
    2. 浏览器解析HTML生成DOM树，解析CSS生成CSS规则树，两者生成渲染树（渲染树与DOM树区别是，渲染里没有head和display为none的节点）
  8. 浏览器布局渲染
    根据渲染树布局，计算CSS样式，显示页面，可能引起重绘和重排，其中重绘是屏幕的一部分重画，不影响整体布局，如改个颜色，重排需要重新计算DOM树，显示最后页面
